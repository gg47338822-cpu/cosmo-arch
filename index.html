L1

Order Intake

订单接入 -- 模板初筛

接收外部订单，验证参数，匹配SKU模板。根据发货点/收货点自动关联VAS站点和航线。
L2

Dispatch Planning

调度规划 -- 模拟推演

多方案推演，评估各VAS/无人机/骑手组合的可行性和成本。选出最优方案。
L3

Resource Allocation

资源分配 -- 因子查询 + 方案拼合

查询注册资源的实时可用状态，拼合A-B-C三段资源包。渐进式锁定，从软锁到硬锁到原子锁。
L4

Execution Monitor

执行监控 -- 实时追踪

监控订单执行全链路。GPS定位、电池电量、ETA滚动更新。OrderTracer追踪系统，6阶段全覆盖。
L5

Exception Handling

异常处理 -- 地面兜底

异常分级响应：轻微→重试，中等→降级(地面兜底)，严重→取消+赔付。电池三层动态阈值保障安全。
L6

Settlement Analytics

结算分析 -- 数据闭环

订单完成后自动结算。多方分账(VAS/OP/骑手平台)。数据分析驱动运营优化。
V

VAS Station

Direct Registration
Registration: Station / Pad / Crew
Dispatch: SP directly schedules
Resources: Landing pad + Energy + Crew = Atomic bundle
Key Rule: Origin VAS bundle includes drone; Destination does not
O

OP Operator

Direct Registration
Registration: Operator / Model / Individual drone
Dispatch: SP selects drone, OP approves flight
Key Rule: Drone ownership at OP, custody at VAS, scheduling at SP
Flight = custody transfer: Origin VAS -> Destination VAS
R

Rider Platform

Platform Registration
Registration: Platform-level (SF / Dada / In-house)
Dispatch: SP broadcasts demand -> Platforms bid -> Winner dispatches internally
Key Rule: SP never selects individual riders
C-segment: ETA-driven pre-dispatch for seamless handoff
